# 旅行资源页面内容优化方案

> 目标：在不改变整体布局风格的前提下，提升“旅行资源”页的信息结构与可读性，让一线执行人员一眼看清当日行程与关键资源。

---

## 页面结构（按天矩阵）

每一天一行卡片（矩阵行），由左到右分为 4 个区块：

1) **日期列**（固定窄列）  
2) **行程概览**（上午/下午/晚上，轻量摘要）  
3) **资源配置**（酒店/车辆/导游/安保/票务）  
4) **餐饮安排**（浅色背景块：早餐/午餐/晚餐）  

> 底部保留“当日特殊备注”文本区（全宽）。

---

## 1. 日期列（左侧）

**内容**
- 日期（MM-DD）
- 周几（如 Tue）
- 团期第几天（如 D3）
- 人数（如 48 人）
- 城市（当前城市 / 目的地）

**展示建议**
- 日期与周几加粗；团期天数与人数作为次级信息。
- 城市两行显示：`所在城市` / `目的城市`。

**空态**
- 未设置城市：显示“未设置城市”。

---

## 2. 行程概览（中列）

**结构**
- 按时段分三行：上午 / 下午 / 晚上
- 每行内容：`主地点` + `核心活动` + `人数`

**样式建议**
- 时段标签使用小圆点或小图标。
- 内容区可一行展示，超长省略。

**空态**
- 未安排：显示“未安排”。

---

## 3. 资源配置（右侧）

建议分成 5 个模块块（卡片内子区）：

1) **住宿酒店**  
   - 酒店名称
   - 地址（可选，弱化显示）
   - 联系人 / 电话

2) **车辆调度**  
   - 车牌 / 车型
   - 司机姓名 / 电话
   - 座位数（可选）

3) **随团导游**  
   - 姓名 / 电话
   - 证件号（可选）

4) **安保人员**  
   - 姓名 / 电话
   - 班次（可选）

5) **票务/预约状态**  
   - 已确认 / 待确认 / 未设置

**展示规则**
- 次要字段（地址/证件号/班次）仅有值时显示。
- 资源缺失时统一显示“未设置”。

---

## 4. 餐饮安排（浅色块）

**结构**
- 早餐 / 午餐 / 晚餐三行
- 每行：`餐厅名称` + `人数` + `状态`

**状态**
- 已确认 / 待确认 / 未设置

**样式**
- 统一浅色背景块（绿色/浅蓝均可）
- 时段标签加小图标（早餐/午餐/晚餐）

---

## 5. 当日特殊备注（底部）

- 简洁的单行或多行备注输入区
- 空态文案：`添加当日特殊备注...`

---

## 文案与状态规范

**状态值统一**
- 已确认
- 待确认
- 未设置
- 变更中（可选）

**空态统一**
- “未设置”替代“请输入/输入XXX”

**交互提示（可选）**
- 资源槽空态：`点击选择...` 或 `拖拽分配...`

---

## 字段优先级（排序建议）

1. 日期 / 周几 / 团期第几天
2. 上午 / 下午行程
3. 住宿酒店 / 车辆调度
4. 导游 / 安保
5. 餐饮安排
6. 特殊备注

---

## 简化示例（单日）

- 左列：`07-01 Tue` / `D1 · 48人` / `深圳 → 香港`
- 行程概览：
  - 上午：`香港大学 · 校园参访 · 48人`
  - 下午：`西九文化区 · 参观 · 48人`
  - 晚上：`未安排`
- 资源配置：
  - 住宿：`港岛酒店` / `张老师 138xxxx`
  - 车辆：`粤B·12345` / `李师傅 139xxxx`
  - 导游：`王导 137xxxx`
  - 安保：`未设置`
  - 票务：`待确认`
- 餐饮：
  - 早餐：`未设置`
  - 午餐：`园区餐厅 · 48人 · 已确认`
  - 晚餐：`未设置`

# Studio / Field 功能设计（移动端团组执行）

> 本文用于恢复并完善“Studio 与 Field”的功能设计说明。
> 说明：Studio 定义为“移动端团组执行工作台”，面向现场执行人员；Field 定义为“现场记录字段体系”。

## 1. 定位与目标
### Studio（移动端）
- 面向一线执行人员在现场查看团组与行程
- 快速记录到达/离开、人数、异常、费用、现场备注
- 低学习成本、强可用性、弱网可用

### Field（字段体系）
- 让现场记录内容可配置、可复用、可追溯
- 支持不同团组类型/不同场景的差异化字段
- 保持历史数据，不随配置变化而丢失

## 2. 角色与场景
### 角色
- 现场执行/领队/导游
- 协调员/运营（查看进度与异常）

### 场景
- 出发/到达确认
- 活动开始/结束打卡
- 人数核对、缺席记录
- 资源使用确认（场地、车辆、酒店等）
- 异常事件记录（延误、变更、事故）
- 费用记录与票据上传

## 3. Studio 关键页面与功能
### 3.1 首页/今日概览
- 今日团组列表（按时间排序）
- 待处理事项（未确认到达、异常待处理）
- 快速入口：签到、记录异常、上传资料

### 3.2 团组详情页
- 行程时间线（上午/下午/晚上）
- 今日重点：资源地点、集合时间、注意事项
- 当前状态：待出发/进行中/已完成

### 3.3 活动/资源卡片
- 快速打卡：开始/结束/到达/离开
- 记录人数与备注
- 现场照片上传（可选）

### 3.4 成员名单
- 签到/缺席/请假
- 人数汇总与差异提示

### 3.5 异常与变更
- 快速记录异常类型（延误/缺席/变更）
- 事件时间、位置、责任人
- 关联行程节点

### 3.6 资料与附件
- 现场照片、文件上传
- 按团组与日期归档

## 4. Field（现场记录字段体系）
### 4.1 字段类型（建议）
- 文本（单行/多行）
- 数值（人数/金额）
- 日期/时间
- 选择（单选/多选）
- 布尔（是/否）
- 附件（图片/文件）
- 地理位置（自动记录）

### 4.2 字段规则
- 必填/可选
- 条件必填（某类异常出现时必填）
- 自动字段：时间戳、定位、操作者

### 4.3 字段绑定
- 与团组节点绑定（例如“上午-活动开始”）
- 与资源/活动绑定（例如“酒店入住确认”）

## 5. 离线与同步
- 弱网/无网可保存草稿
- 恢复网络后自动同步
- 同步失败可重试并提示

## 6. 权限与审计
- 默认仅管理员与执行人员可记录
- 所有记录保留操作人、时间、地点
- 历史记录可追溯、不可覆盖

## 7. UI 设计原则（移动端）
- 任务优先：操作按钮大且集中
- 单手可用：底部固定操作区
- 低干扰：减少二级菜单
- 高可读：卡片式、清晰时间线

## 8. 与其他模块的衔接
- 日历详情：可展示现场记录与异常
- 团组管理：可查看现场状态与备注
- 统计报表：汇总异常、延误、费用
